Pipetting Performance Workflow Instructions



Welcome to the Pipetting Performance Workflow!

This workflow helps you test the pipetting performance of your robot, making sure it works reliably. Here, you can get an overview of it.


You will be guided through the process of:

  1. Selecting the experiment parameters for the performance test.
  2. Running the pipetting performance protocol on your liquid handling system.
  3. Measuring the absorbance on a plate reader.
  4. Downloading an HTML report of the performance of your pipetting robot.


Quick, full or 8-channel run?

Based on your needs, you can select one of three run modes.


The quick run option:

  • Should only be used for 96-channel or 384-channel pipetting heads.
  • Allows you to quickly fill a single plate with Orange G using your robot.
  • Gives a rough overview of the precision of your system using absorbance measurements.

The full run option:

  • Should only be used for 96-channel or 384-channel pipetting heads.
  • Should be used if a more precise, quantitative approach is needed.
  • Requires at least three plates being measured to assess channel deviations and inter plate variability.
  • Includes running an optional gravimetric test to get an overview of the accuracy of the system.

The 8-channel run option (also for single-channel):

  • Should always be used when testing an 8-channel pipetting head. Can also be used for testing single-channel pipettes.
  • Always uses one plate and 12 replicates, as they all fit on one plate.
  • Can assess channel deviations and inter plate variability.
  • Allows running an optional gravimetric test to assess the overall accuracy of the system.

Running the pipetting protocol on your robot

To assess the performance of your system, you need a pipetting protocol for it that pipettes a given amount of water into each well of a microplate and then adds the appropriate amount of Orange G (OG) solution on top of that. Please consider the following points when running such a protocol.

  • For some liquid handling systems, pipetting scripts are already available. Otherwise, a new script can be written according to the guidelines provided above.
  • Make sure you correctly set up your deck according to the instructions of your pipetting script.
  • Use Milli-Q water for the water pipetting step and in your OG solution.
  • Make sure the volume of OG pipetted into each well matches the volume you specify in the workflow.
  • The appropriate OG concentration depends on the volume pipetted. A good starting point is a concentration of 1 g/l.
  • This concentration works well for volumes between 2 and 10 microliters.
  • If your absorbance values end up below 0.1 or above 1, consider running the test again, but with a higher/lower OG concentration, respectively.
  • If a run is repeated or a quick run is followed up by a full run, make sure to use fresh tips. It is also recommended to clean the channels with EtOH before the next use.
  • If you are testing a single-channel head, it makes most sense to just pipette the first row. Make sure to then also only measure the pipetted row(s) on the plate reader.

Please start the script on the robot now! While the robot is running the protocol, please enter your experimental setup in the following steps. Tips on how to measure the plate will follow.



Preparing the plate(s) for measuring the absorbance

After pipetting, the content of your plate(s) will not be perfectly mixed. Consider the following points to make sure your reader measurement is as realiable as possible.

  • Use a shaker to thoroughly mix the water in the wells with the OG solution.
  • Shaking for 1-2 min at 2500 RPM should be sufficient.
  • It is highly recommended to seal the plate with aluminium foil or tape to avoid spilling.
  • Especially for 384-well plates, it is necessary to spin down the liquid in the wells with a centrifuge. Recommended settings are 3 min at 1500 × g.
  • Liquid will get stuck on the foil if the plate is not centrifuged before removing it, making the results unreliable.
  • Visually inspect the plate to make sure no dust or lint is stuck to the bottom of the wells (on the outside).
  • Make sure any foil or lid is removed before putting the plate into the reader.

Measuring the absorbance with the plate reader

The following key points should be considered when measuring the absorbance with your plate reader of choice.

  • Make sure you have the right type of plate selected on the reader.
  • The wavelength should be set to 450 nm for measuring OG absorbance.
  • Measuring once per well should be sufficient. If you make multiple measurements per well, check the SD in the output file to make sure that the results are consistent for each well.
  • If you measure multiple times per well, the mean absorbance for each well will be used for assessing the pipetting performance.
  • For some readers, scripts with the appropriate measurement settings are already available.

NOTE: If you did not pipette a whole plate, only measure the row(s) you pipetted to avoid problems during data analysis.


Uploading the reader output file(s)

If you selected the option, the gravimetric accuracy test will be performed in the next step. Afterwards, the reader files can be uploaded. Please consider the following points before creating the files.

  • If you have multiple plates, each one should be saved in a separate file.
  • For a list of supported formats, go to the file upload step.


Running the gravimetric accuracy test

You selected the additional gravimetric accuracy test, which is used to find out if the whole pipetting head (all channels on average) pipette the right volume. If the individual channels give good results in the absorbance test but the gravimetric test is off, there might be a problem with the overall volume pipetted by all channels (the channels pipette too little or too much in general).


This test uses a separate protocol on the robot and only a single plate. The robot should pipette a given amount of water into each well of the plate.


  • Work quickly to avoid evaporation and enter the weight as precisely as possible.
  • For a 96-well plate, use a volume between 50 and 100 μl per well to get meaningful results.
  • If the resulting volume per well is a bit lower than expected, that might be normal, especially when working with low volumes.

General steps:

  • Measure the weight of the plate before the first round of pipetting.
  • After the fist round of pipetting, quickly weigh the plate and enter the new weight.
  • Keep the water in the plate and continue with the second round of pipetting.
  • Weigh again and enter the second weight measurement.
  • Do the same for the third and final measurement.


The following reader file types are supported:

  • CSV file in plate format with labeled rows and columns (*.csv)
  • Tecan reader Excel sheet (*.xlsx)
  • Molecular Devices SpectraMax XML export (*.xml)

Additional technical information about the file types:

  • CSV files are simply converted to a pandas DataFrame of the same shape. If they don't have the format of the plate, all graphs will look wrong and the results will be misleading.
  • For Tecan reader Excel files, either the first cell of column 1 with "<>" in it is used to find the absorbance table or if that cannot be found, the first cell of column 1 with "Mean" in it is found to allow reading the mean for each well in case of multiple measurements per well. If there are multiple tables, the first one is used.
  • For SpectraMax XML files, the "RawData" for each well is used. Other SpectraMax output files are not supported.


If an unsupported file is uploaded, there will be an error!


If you encounter an error with your uploaded data, you can try to use one of the Excel templates provided in the repository.



Downloading and interpreting the test results

In this step, you can download the HTML report and take a look at your results. If you are not sure how to interpret them, read on:

  • The report starts with an overview of the tested parameters. This should help when comparing reports.
  • Then, a brief overview of the results of the test criteria is given with red indicating major problems with the tested head, orange indicating minor problems and green indicating that all test criteria were passed.
  • This overview does not provide a full picture! Please take a look at the diagrams below as well.
  • The graphs and heatmaps below show additional information that might be useful for finding out what the problem is.
  • The second page contains the report for the gravimetric test (if it was performed).
  • The following pages contain diagrams for the individual plates (if replicates were used).

It is always recommended to have a thorough look at all the diagrams. The criteria can be passed even if there is a problem present with one of the channels.


NOTE: When testing a single-channel head, please look at the row corresponding to the channel you measured in the plots. If you measured more wells than were pipetted, your results will look wrong, even if they are perfectly fine. Also, column averages will not be displayed correctly. Please ignore them.


Make sure to download the report before finishing this step. More files can be found in the "Files" tab.
